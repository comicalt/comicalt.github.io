0x01 eval和assert函数

概念

eval() —–> 把字符串作为php代码执行(注意这里的代码执行是php代码执行，而不是命令执行)

assert()—> 传入一个assertion，如果assertion是字符串，它会被当做php代码执行(断言功能暂时不谈)

官方手册 eval

官方手册assert

differ_1:代码模式

从官方手册我们可以看到eval函数只有一个参数，即需要被执行的字符串代码

但是代码不能打开/关闭PHP标签，但可以用合适的PHPtag 来离开、重新进入PHP模式

该模式的重要之处是 ?> <? 前面第一个?>用来闭合 而后面那个则用来新开启一个php模式

而assert不支持这种模式

differ_2:return

return 语句会立即中止当前字符串的执行。

assert也不支持这种模式

differ_3:返回值不同

eval() 返回 NULL，除非在执行的代码中 return 了一个值，函数返回传递给 return 的值。

可以看到第一个返回的是空，而assert一直返回的true

differ_4:本身性质不同

eval —> 本身是一个语言构造器而不是一个函数，因此不能被可变函数调用

assert –> 在7之前的版本中assert是一个函数，而在7中其变成了一种语言结构

通配符

?：匹配任何一个字符

*：匹配任何字符串/文本，包括空字符串

[.....]：匹配范围中任何一个字符

[a-z]：匹配a-z范围中任何一个字符

{a,b}：对以逗号分割的文件列表进行拓展，进行两次命令

{...}与[...]有一个很重要的区别。如果匹配的文件不存在，[...]会失去模式的功能，变成一个单纯的字符串，而{...}依然可以展开。
